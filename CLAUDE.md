## Project Overview

Personal portfolio website for Aleksander Zubala (zubala.com), built with Hugo static site generator and hosted on GitHub Pages.

## Architecture

### Directory Structure

- **`hugo/`** - Hugo source files (content, configuration, themes)
  - `hugo/content/` - Markdown content files (about page, projects)
  - `hugo/config.toml` - Site configuration (theme, menus, social links, metadata)
  - `hugo/themes/hugo-goa/` - The hugo-goa theme
  - `hugo/static/` - Static assets (images, icons)
- **Root directory** - Built/published site (generated by Hugo)
  - This is the GitHub Pages publish directory
  - Files here are auto-generated and should not be edited directly

### Build Process

Hugo builds from the `hugo/` directory and publishes output to the parent directory (root). The root directory is served by GitHub Pages at `https://zubala.com`.

**Important**: All content changes must be made in `hugo/` directory, never in the root.

## Common Development Commands

### Building the Site

```bash
# Build site (from repository root)
cd hugo && hugo

# Build with drafts included
cd hugo && hugo -D

# Serve locally with live reload (default: http://localhost:1313)
cd hugo && hugo server

# Serve with drafts
cd hugo && hugo server -D
```

### Content Management

```bash
# Create new project page
cd hugo && hugo new content/projects/project-name.md

# Create new content with custom path
cd hugo && hugo new content/path/to/file.md
```

### Content Structure

Project pages in `hugo/content/projects/*.md` use this front matter structure:

```toml
+++
title = "Project Title"
tags = ["Tag1", "Tag2", "Tag3"]
draft = false
date = "YYYY-MM-DDTHH:MM:SS+01:00"
showpagemeta = true
categories = ["project"]
comments = false
showcomments = false
+++
```

## Site Configuration

The site is configured via `hugo/config.toml`:
- Base URL, language, author info
- Theme selection (hugo-goa)
- Menu structure (about, projects)
- Social media links (GitHub, LinkedIn, Twitter, etc.)
- Google Analytics, metadata
- Copyright year (update annually)

## Deployment

This site uses GitHub Pages with a custom domain (zubala.com). The deployment process:

1. Make content changes in `hugo/` directory
2. Build the site: `cd hugo && hugo`
3. Commit both source files and generated files
4. Push to GitHub: `git push origin master`

The root directory must contain the built site for GitHub Pages to serve correctly.

## Theme Information

Uses the **hugo-goa** theme, a minimal personal website theme. Theme files are in `hugo/themes/hugo-goa/`. To customize layout or styling, create overrides in `hugo/layouts/` or `hugo/static/css/`.
